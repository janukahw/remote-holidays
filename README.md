# Out of Range

**🌐 Live: https://janukahw.github.io/remote-holidays/**

A static site that ranks remote UK places to stay — cottages, cabins and campsites — by how cut off they are: no WiFi, no mobile signal, off-grid power, miles from the nearest road. The defining attribute of every place is its connectivity.

> **577 places** from multiple providers — **National Trust** (439), **Unplugged** (44), **Unyoked** (43), **Canopy & Stars** (8) and **Cool Places** (43) — sorted by remoteness score.

## Stack

Plain HTML, CSS, and vanilla JavaScript. **No build step for the site, no dependencies, no server required** — open `index.html`, or host the folder anywhere static (GitHub Pages, Netlify, Cloudflare Pages).

The only network dependencies are the Google Fonts link (Fraunces + Inter, with system fallbacks) and the property photos, which are hotlinked from each provider's CDN.

## The data

`js/data.js` is **generated** from the spreadsheets in `data/` by `tools/build_data.py`. Each provider has its own column layout; the build script normalises them into one schema, merges, sorts by score (highest first), and features the top few of each provider for the homepage.

The site keeps data close to source: each place stores its raw `signals` array and a `wifiListed` flag, and `js/site.js` *derives* the connectivity badges (No WiFi / Patchy WiFi / No signal / Off-grid power) and feature filters (island, bothy, dark skies, lighthouse, hard-to-reach) from them.

Some traits are universal to a provider rather than per-place — every Unplugged cabin is off-grid, Wi-Fi-free and ships a phone lockbox by design — so the build injects those signals for that provider (see `universal_signals` in `build_data.py`).

### Remoteness score (unified 0–100)

`score` is a **unified 0–100 rating applied across all providers**, so they are directly comparable. It's the sum of seven capped dimensions: access (25), mobile signal (20), off-grid services (13), WiFi (12), isolation framing (12), distance to amenities (10), wild setting (8). Each place also carries `scoreBreakdown` (per-dimension points, shown on its detail page) and a one-line `scoreWhy`.

Scores live in `data/unified_scores.json`, merged in by `build_data.py` (falls back to the source score for any id not present). That file was generated by re-reading every listing's scoreable fields against the rubric above and summing the breakdown. Regenerate it only when the rubric or listings change. Scoring reflects how each place is *described* in its listing, not measured GPS isolation.

### Adding a provider

1. Drop the new `<provider>_remoteness_ranking.xlsx` in `data/`.
2. Add a `SOURCES` entry in `tools/build_data.py` mapping its columns to the logical fields (and any `universal_signals`).
3. Re-run:

```powershell
# one-time: pip install openpyxl
python tools/build_data.py
```

Nothing in the site needs to change — the Provider filter, counts and facets update from the data.

## Run it

Open `index.html` directly (no server needed), or for a local server:

```powershell
python -m http.server 8000   # then visit http://localhost:8000
```

### Cache-busting

Asset links carry a content-hash query (`css/styles.css?v=<hash>`) so browsers refetch a file only when it changes. After editing **css/js** by hand, restamp with:

```powershell
python tools/bump_version.py
```

`build_data.py` runs this automatically, so a data rebuild restamps too. (Tip: during rapid local edits, hard-refresh with Ctrl+F5 if you don't see a change.)

This is an independent, non-commercial guide and is **not affiliated with or endorsed by** them. **Check each provider's terms of use before deploying this publicly** (it hotlinks their images and republishes listing text).

- "No signal" cuts both ways: you can't be reached in an emergency either. This is a planning aid, not a safety device.
